# Kubexm 测试用例

## 1. 全局与通用命令测试
| 用例编号 | 命令 | 描述 | 预期结果 |
|---------|------|------|---------|
| GL-01 | `kubexm version --verbose` | 测试 verbose 全局日志标识 | 输出 Debug 级别的详细日志信息 |
| GL-02 | `kubexm --yes cluster create -f conf.yaml` | 测试自动确认标识 | 全局跳过用户交互询问，直接执行创建流程 |

## 2. 集群生命周期管理测试 (Cluster Commands)
| 用例编号 | 命令 / 参数 | 描述 | 预期结果 |
|---------|-------------|------|---------|
| CL-01 | `cluster create -f conf.yaml --skip-preflight` | 跳过环境预检直接安装集群 | 系统忽略基础连通性和预检，直接生成并下发任务 |
| CL-02 | `cluster create -f conf.yaml --dry-run` | 模拟创建集群 | 规划并输出 DAG 执行流，但不对宿主机做任何修改 |
| CL-03 | `cluster delete -f conf.yaml --force` | 强制删除集群 | 无需二次确认，安全排空节点并卸载相关组件及配置 |
| CL-04 | `cluster upgrade -f conf.yaml -t v1.24.3` | 集群版本升级 | 控制面和 Worker 节点平滑升级到指定版本 |
| CL-05 | `cluster upgrade -f conf.yaml` | 缺少版本号升级 (负面测试) | 返回错误提示，要求必须指定 `-t / --version` 参数 |
| CL-06 | `cluster backup -f conf.yaml -t all` | 备份全部集群数据 | 在默认/指定目录生成涵盖 PKI, ETCD, Kubernetes 配置的备份包 |
| CL-07 | `cluster restore -f conf.yaml -b /p/b.tar.gz -s /p/e.db` | 集群全量恢复 | 校验快照路径无误后，停服覆盖数据并重启相关组件恢复业务 |
| CL-08 | `cluster health -c apiserver` | 健康状态检测 | 仅输出 APIServer 的健康检查结果 |
| CL-09 | `cluster reconfigure -f conf.yaml -c kubelet --restart`| 重新配置 kubelet | kubelet 配置文件更新，并且重启 kubelet 服务 |

## 3. 节点管理测试 (Node Commands)
| 用例编号 | 命令 / 参数 | 描述 | 预期结果 |
|---------|-------------|------|---------|
| NO-01 | `node list -c mycluster --kubeconfig /p/config` | 打印节点列表 | 正确解析 kubeconfig，输出集群内所有节点及其状态 |
| NO-02 | `node get node-1 -c mycluster -o json` | 获取具体节点信息 | 返回 JSON 格式的 node-1 详尽配置信息 |
| NO-03 | `node cordon node-1 -c mycluster` | 封锁节点 | 节点被成功打上 Unschedulable 标记 |
| NO-04 | `node uncordon node-1 -c mycluster` | 解封节点 | 节点恢复为正常可调度状态 |
| NO-05 | `node drain node-1 -c mycluster --force` | 排空节点 | 忽略 DaemonSet，强制驱逐所有 Pods 并使节点下线 |

## 4. 证书管理测试 (Certs Commands)
| 用例编号 | 命令 / 参数 | 描述 | 预期结果 |
|---------|-------------|------|---------|
| CE-01 | `certs renew all --yes` | 一键更新所有证书 | Kubernetes 和 ETCD 的 CA 及 Leaf 证书全量更新且组件重启完毕 |
| CE-02 | `certs renew kubernetes-certs` | 仅更新 K8s 业务证书 | K8s 的 leaf 证书到期时间延后，原 CA 证书体系保持不变 |
| CE-03 | `certs check-expiration` | 查看证书过期情况 | 准确输出各节点各项证书的有效起止时间和剩余天数 |

## 5. 镜像仓库及离线资源测试 (Registry & Utilities)
| 用例编号 | 命令 / 参数 | 描述 | 预期结果 |
|---------|-------------|------|---------|
| UT-01 | `registry create --yes` | 搭建私有镜像仓库 | 在指定节点成功部署并启动容器 Registry 服务 |
| UT-02 | `download -f conf.yaml -o /tmp/packages` | 离线资源打包下载 | 忽略 host 联通性检查，拉取所有的 binaries, images 到指定目录 |
| UT-03 | `iso create -f conf.yaml -o output.iso` | 制作离线安装光盘 | 最终生成一个包含必要系统引导和 Kubernetes 资源的 ISO 镜像文件 |

## 6. 架构强约束与边缘用例测试
| 用例编号 | 测试目标 | 描述 | 预期结果 |
|---------|---------|------|---------|
| AR-01 | **Step 幂等性** | 第一次成功执行 `cluster create` 后，不清理环境马上执行第二次 | 执行流中所有具备检测逻辑的 Step 应当快速跳过 (Skip)，且系统无异常 |
| AR-02 | **中心机器约束** | 修改 `host.yaml` 中的机器 IP 为 `127.0.0.1`，执行安装 | Preflight 阶段抛出严重错误，禁止使用本地环回地址装机 |
| AR-03 | **离线模式自动感知**| 将 `download` 生成的 `packages` 目录放入工作区，拔掉网线 | 工具在创建集群时感知到 `packages` 目录存在，全程不报错并安装成功 |
| AR-04 | **异常恢复机制** | 故意填写一个不存在的 SSH 密钥路径 | Connector 报错，Runner 捕获错误并触发执行终止，返回清晰的提示 |
| AR-05 | **避免代码 Panic** | 故意制造一个会导致 Module Plan 阶段空指针的异常 | 被 `SafePlan` 包装器接管，打出 Error 日志，安全退出，而非 Stacktrace |

| 用例编号 | 测试目标 | 执行指令示例 | 前置条件 | 验证手段 / 预期结果 |

| **AR-002** | 全局详细日志输出 | `kubexm --verbose version` | 无 | 终端能捕获到底层的 Debug 级别日志打印。 |
| **AR-003** | 不产生环境副产物 | 任意 `kubexm [cmd] --dry-run` | 无 | 观察屏幕打印出 `ExecutionGraph` 逻辑树。登录靶机确认没有任何包上传、没有 systemctl 改动。 |

## 2. 核心集群全生命周期用例 (Cluster Commands)

### 2.1 Cluster Create (创建)
| 用例编号 | 参数组合 / 指令 | 测试重点与步骤 | 预期结果断言 |
|---|---|---|---|
| **CL-CRE-01** | `cluster create -f M01.yaml` | 在线状态部署标准集群 | 系统探测网络可达性，拉取在线资源并走完全部组装 Step。K8s 各系统组件 Running。 |
| **CL-CRE-02** | `cluster create -f M16.yaml` | 带外部高可用组合的创建 | 特别关注 `LoadBalancerModule` 编排。`keepalived` 主备生效并正确暴露 VIP。 |
| **CL-CRE-03** | `cluster create -f M02.yaml` | 全二进制纯净部署 | 检查 ETCD 和 APIServer 的 systemd unit 启动成功，互信 TLS 配置有效。 |
| **CL-CRE-04** | `cluster create -f conf.yaml --skip-preflight` | 忽略预检查创建 | 跳过 SSH 初步验证与磁盘内存空间校验，强制生成部署流。 |
| **CL-CRE-05** | `cluster create -f conf.yaml` (离线自动感知) | 把断网机上的 `packages/` 置于目录 | 成功检测离线标记，无任何外网报错，集群构建成功。 |

### 2.2 Cluster Delete (销毁)
| 用例编号 | 参数组合 / 指令 | 测试重点与步骤 | 预期结果断言 |
|---|---|---|---|
| **CL-DEL-01** | `cluster delete -f conf.yaml` | 交互式集群卸载 | 逐步反向（逆序清理），确认每个节点上的残留文件与 iptables 路由表恢复至原生状态。 |
| **CL-DEL-02** | `cluster delete -f conf.yaml --force` | 无脑强删集群 | 忽略节点 drain 过程中部分驱逐等待时间，暴力清除 Kubelet 及 Docker/Containerd。 |

### 2.3 Cluster Upgrade (升级)

### 2.4 Cluster Backup & Restore (备份恢复)

## 3. 节点维护操作用例 (Node Commands)

| 用例编号 | 参数组合 / 指令 | 测试重点与步骤 | 预期结果断言 |
|---|---|---|---|
| **NO-001** | `node list -c mycluster --kubeconfig /k/conf` | 列出节点概览表 | 工具使用指定集群凭证请求 API Server 返回列表信息。 |
| **NO-002** | `node get node-1 -c mycluster -o json` | 拉取特定节点详单 | 将指定节点的完整注册及资源视图通过 JSON 对象输出。 |
| **NO-003** | `node cordon node-1 -c mycluster` | 修改节点可调度属性 | 将对象置为 SchedulingDisabled。 |
| **NO-004** | `node uncordon node-1 -c mycluster` | 解除节点调度封印 | 恢复至节点可负载业务。 |
| **NO-005** | `node drain n-1 -c myc --force --ignore-daemonsets` | 腾退节点且无视孤儿 Pods | 强制疏散该宿主机上的实例，直到宿主机排空可以安全关机下线。 |

## 4. 证书系统轮换用例 (Certs Commands)

| 用例编号 | 参数组合 / 指令 | 测试重点与步骤 | 预期结果断言 |
|---|---|---|---|
| **CE-001** | `certs renew all --yes` | 整体集群 TLS 过期延保 | Root CA 及叶子级证书全部重新颁发，自动同步到所有对应节点，并触发组件热重启以加载新证书。 |
| **CE-002** | `certs renew kubernetes-ca --yes` | 专属翻新 K8s Root CA | 旧叶子证书连带被废除重签，K8s 通信链焕然一新，而 ETCD 不受波及。 |
| **CE-003** | `certs renew etcd-certs --yes` | 叶子级静默热更新 | 根证书信任链不变的情况下更新 ETCD Node 的 Leaf 证书。 |
| **CE-004** | `certs check-expiration` | 核查过期时间 | 正确拉取各目录下的 PEM 证书并计算出各自还余留的天数/小时数。 |
